## Format Specifications Implemented

### Claude Code Format (JSON)
```json
{
  "mcpServers": {
    "server-name": {
      "type": "stdio",
      "command": "npx",
      "args": ["-y", "package"],
      "env": { "KEY": "value" }
    }
  }
}
```

### Gemini Format (JSON)
```json
{
  "mcpServers": {
    "server-name": {
      "command": "npx",
      "args": ["-y", "package"],
      "env": { "KEY": "value" }
    }
  }
}
```

### Codex Format (TOML)
```toml
[mcp_servers.server-name]
command = "npx"
args = ["-y", "package"]
startup_timeout_ms = 20000

[mcp_servers.server-name.env]
KEY = "value"
```

## Key Features

### 1. Intelligent Field Mapping
- **Claude Code ‚Üí Codex**: Strips `type` field, adds `startup_timeout_ms` (default: 20000), converts url to url
- **Codex ‚Üí Claude Code**: Adds `type: "stdio"` field (or "http" for url-based), strips `startup_timeout_ms`
- **Claude Code ‚Üí Gemini**: Strips `type` field, converts url to httpUrl for HTTP servers
- **Gemini ‚Üí Claude Code**: Adds `type: "stdio"` field (or "http" for httpUrl), converts httpUrl to url
- **httpUrl/url detection**: Automatically determines server type (stdio vs http) based on fields present

### 2. Edge Case Handling
- ‚úÖ Servers with no args (empty array)
- ‚úÖ Servers with no env (empty object)
- ‚úÖ Multiple servers in single config
- ‚úÖ Env variables with special characters (URLs, paths)
- ‚úÖ Same-format passthrough (no conversion needed)

### 3. HTTP/SSE Server Support
- ‚úÖ Stdio servers (command/args based)
- ‚úÖ HTTP servers (url based for Claude/Codex, httpUrl for Gemini)
- ‚úÖ SSE servers (same as HTTP)
- ‚úÖ Proper type field detection and conversion

### 4. Error Handling
- Validates JSON/TOML structure
- Throws clear errors for:
  - Malformed JSON/TOML
  - Missing required fields (`mcpServers` or `mcp_servers`)
  - Unsupported formats
